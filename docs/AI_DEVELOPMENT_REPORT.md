# AI Development Report - AI Email Generator

## Инструменты и модели

Для разработки использовал Claude Code - это CLI-инструмент от Anthropic, работает прямо в терминале как пара-программист. Основная модель - claude-sonnet-4-6, для более сложных задач типа архитектуры AI-слоя и настройки CI подключал claude-opus-4-8.

Сам продукт тоже использует Claude: `AnthropicProvider` вызывает claude-sonnet-4-6 для генерации писем. То есть AI использовался и для написания кода, и является частью самого приложения.

---

## Как строился процесс

Разбил проект на 7 фаз, каждая в отдельном markdown-файле (00-setup.md, 01-landing.md и т.д.). В каждом файле - конкретные промпты с описанием того, что нужно сделать. Это помогало не терять контекст между сессиями и понимать, на каком этапе находишься.

Порядок фаз: Setup - Landing - Auth - Dashboard + AI - Pricing - Profile - Polish - Docs/Deploy.

Ошибки всплывали тремя способами: TypeScript ловил часть до запуска (`npx tsc --noEmit`), остальное обнаруживалось при визуальной проверке в браузере или когда что-то не работало при тестировании вручную.

## Ключевые промпты по фазам

### Setup

Начал с одного большого промпта на инициализацию - Next.js 14, TypeScript strict, Tailwind, shadcn/ui, Supabase с cookie-сессиями, Anthropic SDK, framer-motion. Сгенерировалось почти всё сразу, кроме мелочи: npm не принял название пакета с пробелами.

Второй момент - Tailwind не работал. Оказалось, кастомный `postcss.config.mjs` перетирал дефолты Next.js и терял autoprefixer. Дописал его явно, стили подхватились.

### Landing

Попросил сделать Hero с анимациями через framer-motion. После рендера контент оказался невидим - `initial={{ opacity: 0 }}` отрисовывался на сервере и так и оставался. Исправил через паттерн с `useState(false)` + `useEffect`: анимация запускается только после mount на клиенте.

### Auth

Промпт был на signup/login/logout через Supabase Auth с SSR-куками. Auth заработал, но регистрация вела себя странно - страница просто обновлялась без редиректа.

Проблема оказалась в том, что в Supabase было включено подтверждение email. `signUp` возвращает `{ user, session: null }` без ошибки, я делал `redirect("/dashboard")`, там сессии нет - редирект на `/login`, и всё по кругу. Добавил проверку `if (data.session === null)` и стал возвращать нормальное сообщение пользователю. Заодно отключил email confirmation в настройках Supabase для разработки.

### Dashboard + AI

Сначала сделал AI-слой отдельно от UI - интерфейс `EmailProvider`, два провайдера (mock и Anthropic), фабрика `getEmailProvider()` читает `AI_PROVIDER` из env. Благодаря этому весь остальной код не знает, какой провайдер используется.

Потом UI дашборда: форма с textarea и двумя Select, API-роут с Zod-валидацией, сохранение результата в БД, история в сайдбаре. После генерации вызывал `router.refresh()` - он перезапускает серверные компоненты и передаёт свежую историю, но клиентский state (результат, форма) не сбрасывается.

### Pricing

Три плана, для гостей кнопки ведут на /signup, для залогиненных открывается модалка с mock-checkout. Server Action `upgradeToPro()` просто пишет `plan = 'pro'` в базу. Badge с планом появился в шапке.

При обновлении UserMenu случайно удалил ссылку на /profile - заметил при ручной проверке, вернул.

### Profile

Страница с email (read-only), датой регистрации, планом, редактируемым именем и счётчиком генераций. Профиль и счётчик загружаются через `Promise.all`, чтобы не ждать последовательно. Кнопка сохранения имени активна только если имя изменилось.

### Polish

Добавил error boundaries на каждый роут (`error.tsx` + `loading.tsx`) и глобальный `not-found.tsx`. Сделал утилиту `normalizeError()` - она приводит любую ошибку (сетевую, JWT, RLS, AI, rate limit) к читаемой строке для пользователя.

Написал unit-тесты на Vitest: 8 кейсов для `normalizeError`, 4 для MockProvider, 5 для Zod-схемы API. Итого 17 тестов, все зелёные. E2E на Playwright - публичные страницы и 404.

Dockerfile с multi-stage сборкой, docker-compose, GitHub Actions с lint - typecheck - test - build. CI работает без реальных ключей, подставляет AI_PROVIDER=mock и placeholder-значения для Supabase.

## Что бы добавил с большим временем

Stripe. Сейчас `upgradeToPro()` пишет план напрямую в базу - это demo flow. Реальная интеграция: заменить action на создание Checkout Session, webhook на `checkout.session.completed` обновляет план. UI трогать не нужно.

Стриминг. Сейчас генерация блокирует UI на 2-4 секунды. Anthropic SDK поддерживает `.stream()`, Next.js умеет `StreamingTextResponse` - текст бы появлялся посимвольно.

Rate limiting. Free-пользователи ничем не ограничены. Нужна проверка в `/api/generate` - если превысил дневной лимит, вернуть 402 с предложением апгрейда.

Ещё провайдеры. Архитектура готова: добавить `OpenAIProvider` или `GeminiProvider` - один файл и одна ветка в фабрике. Можно дать пользователю выбор модели в профиле.

Шаблоны. Предустановленные темы (cold outreach, follow-up, жалоба) - не надо думать, что написать в поле темы.

Нормальные e2e. Сейчас Playwright покрывает только публичные страницы. Нужны тесты с авторизацией: signup - генерация - проверка истории - апгрейд плана. Потребует отдельного тестового Supabase-проекта.

Observability. Sentry для JS-ошибок в продакшене, логирование вызовов провайдера с latency и количеством токенов.
